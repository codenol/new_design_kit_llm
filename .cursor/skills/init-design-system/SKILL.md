# Инициализация проекта на дизайн-системе

## Когда применять

Пользователь просит **«Инициализируй проект»** или «собери витрину компонентов» в проекте, содержащем папку `design-system` (токены, layout, uikit) и ожидает один экран с максимально полным показом компонентов.

## Цель

Создать или обновить главный экран приложения (витрину), на котором представлены:

- Layout с сайдбаром и основной областью
- Хлебные крошки, заголовок страницы
- Панель фильтров: текстовые поля, выпадающие списки, кнопки (в т.ч. иконки), переключатели (InputSwitch)
- Таблица с сортируемыми колонками, чекбоксами выбора, иконкой настроек; в ячейках — бейджи разных цветов и статус-индикаторы (ok, warning, critical, info)
- Хотя бы одно открывающееся меню (контекстное или выпадающее)

## Шаги

1. **Проверить точку входа и стили**
   - В `src/main.tsx` (или аналоге) должен рендериться корневой компонент с Router при необходимости.
   - В `src/App.scss` (или главном SCSS) должны подключаться в порядке: PrimeReact CSS, затем design-system: tokens, helpers, layout, overrides, затем uikit (tokens, styles, Grid, Tree, Message, DialogTabs, StatusBadge).

2. **Layout**
   - Обёртка: `div.layout-wrapper` с `div.layout-main-container` внутри.
   - Сайдбар: `div.layout-sidebar` с логотипом/названием (`.layout-logo`), списком пунктов меню (`.layout-menu`), активный пункт — ссылка с классом `router-link-exact-active` (или NavLink isActive). Внизу сайдбара — пункт «О программе».
   - Контент: `div.layout-main` с отступами; внутри — хлебные крошки (если есть uikit CustomBreadCrumb — использовать его), заголовок H1, панель фильтров, таблица.

3. **Панель фильтров**
   - Несколько InputText (или аналог), один-два Dropdown, кнопки: основная (primary), вторичная (secondary), иконка (например, скачать, настройки). Один-два InputSwitch для переключателей. Расположить в одну строку с отступами (flex, gap).

4. **Таблица**
   - PrimeReact DataTable с колонками: чекбокс выбора, текст (например «Название», «Версия»), колонка с бейджами (разные варианты: ok, warning, critical, info), колонка со статусами. Заголовки — сортируемые где уместно. В шапке — чекбокс «выбрать все» и иконка настроек (Button icon). Данные — моковый массив (5–10 строк).

5. **Меню**
   - Добавить к одной из кнопок или к строке таблицы контекстное меню (ContextMenu) или выпадающее (Menu) с пунктами (например «Редактировать», «Удалить»), чтобы при открытии были видны стили меню.

6. **Тема**
   - По умолчанию оставить системную тему (prefers-color-scheme). При необходимости добавить переключатель, вешающий на `document.body` класс `layout-theme-light` или `layout-theme-dark`.

## Компоненты и классы

- Сайдбар и меню: классы из design-system/layout (`.layout-sidebar`, `.layout-menu`, `.router-link-exact-active`).
- Бейджи: uikit StatusBadge с пропом `code` (ok, warning, critical, info, …) или span с классами `status-badge`, `status-badge--ok` и т.д.
- Кнопки: PrimeReact Button (label, icon, severity), для акцента — класс `p-button-pr` или primary.
- Таблица: DataTable, Column с sortable, selectionMode="multiple", header с Checkbox и кнопкой-иконкой.

## Результат

Один экран, открывающийся по корневому пути (например `/`), с полным layout, фильтрами, таблицей с моковыми данными и открывающимся меню. Без реального API; все данные локальные.
