# Create Page

## Instructions

Создай страницу, которую хочет пользователь.
Если пользователь не сказал какую хочет страницу и не описал её, то сначала спроси и только потом выполняй последовательность ниже.

## Последовательность

1. Создай файл типа Markdown
2. В созданном файле опиши логику блоков страницы. Пользуйся только готовыми блоками, лаяутом и дизайн-системой проекта. Это крайне важно! Не изобретай новые компоненты!!!
3. В соответствии с файлом из пункта 2 создай код для страницы
4. Проверь себя по коду
5. Запусти сборку проекта. Проверь на ошибки. Не смотри на предупреждения. Если сборка собралась, то хорошо. Если есть ошибки — исправь их и переходи к пункту 4.
6. Запусти проект через npm run dev и открой в браузере через mcp playwright.
7. Проверь, что всё из файла в пункте 2 работает. Можно открывать страницы, кликать, заполнять формы, делать скриншоты. Если что-то не работает, то исправь и перейди на пункт 5.

Не забудь — новая фича должна работать в соответствии с UXUI дизайн-системы.
Нельзя создавать TODO. Запрещено создавать TODO.
Никаких todo list!!!

Не забудь — новая страница должна быть добавлена в сайдбар.
Чтобы пользователь мог её быстро найти и перейти по ней.
