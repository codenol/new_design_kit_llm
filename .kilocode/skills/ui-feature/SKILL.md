# UI Feature

## Instructions

Последовательно выполняя все шаги из этого файла сделай интерфейс, который хочет пользователь.
ВАЖНО: НЕ СОЗДАВАЙ TODO, Пользуйся этими шагами ниже.
Если пользователь не сказал какую хочет страницу и не описал её, то сначала спроси и только потом выполняй последовательность ниже.

## Последовательность

1. Создай файл типа Markdown
2. В созданном файле опиши логику блоков фичи. Пользуйся только готовыми блоками, лаяутом и дизайн-системой проекта. Это крайне важно! Не изобретай новые компоненты!!!
3. Проверь, чтобы фича была качественно вписана в лаяут текущей страницы. Если ты не знаешь для какой страницы фича, спроси адрес URL у пользователя.
4. Дай пользователю время посмотреть созданный файл из пункта 1. Если пользователь согласен с такой реализацией, то иди к пункту 5. Если пользователь не согласен, то задавай ему вопросы (пронумеруй их) и переходи с ответами к пункту 2.
5. В соответствии с файлом из пункта 2 создай код для страницы
6. Проверь себя по коду
7. Запусти сборку проекта. Проверь на ошибки. Не смотри на предупреждения. Если сборка собралась, то хорошо. Если есть ошибки — исправь их и переходи к пункту 6.
8. Запусти проект через npm run dev и открой в браузере через mcp playwright.
9. Проверь, что всё из файла в пункте 2 работает. Можно открывать страницы, кликать, заполнять формы, делать скриншоты. Если что-то не работает, то исправь и перейди на пункт 6.

Не забудь — новая фича должна работать в соответствии с UXUI дизайн-системы.
Нельзя создавать TODO. Запрещено создавать TODO.
Никаких todo list!!!
