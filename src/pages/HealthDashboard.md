# Health Dashboard — Страница мониторинга здоровья компонентов

## Описание

Health Dashboard — это страница для мониторинга здоровья компонентов системы в реальном времени. Страница отображает общий статус системы, сетку компонентов с их метриками, раздел предупреждений и позволяет фильтровать/поискать компоненты.

## Структура страницы

### 1. Верхняя панель (Header)

**Компоненты:**
- `StatusBadge` — общий статус системы (healthy/degraded/unhealthy/unknown)
- `LastUpdated` — время последнего обновления данных
- `RefreshButton` — кнопка обновления данных

**Расположение:**
- Flex-контейнер на всю ширину
- StatusBadge слева, LastUpdated по центру, RefreshButton справа

### 2. Основной контент (Main Content)

#### 2.1 Сетка компонентов (Component Grid)

**Компоненты:**
- `DataTable` — отображение компонентов в виде таблицы
- `StatusBadge` — статус каждого компонента
- `FilterPanel` — панель фильтрации и поиска

**Колонны таблицы:**
- `name` — название компонента
- `type` — тип компонента (database, cache, queue, api)
- `status` — статус (с StatusBadge)
- `lastCheck` — время последнего обновления
- `actions` — действия (просмотр деталей)

**Фильтры:**
- Чекбоксы для фильтрации по статусу (healthy, degraded, unhealthy, unknown)
- Чекбоксы для фильтрации по типу (database, cache, queue, api)
- Input для поиска по названию компонента
- Кнопка сброса фильтров

#### 2.2 Модальное окно деталей компонента (Component Details Modal)

**Компоненты:**
- `Dialog` — модальное окно
- `DataTable` — история метрик
- `KeyValueItem` — отображение метрик
- `Message` — отображение ошибок

**Контент:**
- Заголовок с названием компонента и типом
- Метрики:
  - `cpuUsage` — использование CPU (%)
  - `memoryUsage` — использование памяти (%)
  - `diskUsage` — использование диска (%)
  - `networkLatency` — задержка сети (ms)
  - `activeConnections` — активные соединения
- Последние ошибки (если есть)
- Кнопка закрытия

### 3. Раздел Alerts (Предупреждения)

**Компоненты:**
- `DataTable` — список активных предупреждений
- `StatusBadge` — статус предупреждения (active/resolved)
- `Message` — отображение критических предупреждений

**Колонны таблицы:**
- `type` — тип (critical/warning/info)
- `component` — затронутый компонент
- `message` — сообщение об ошибке
- `timestamp` — время возникновения
- `status` — статус (active/resolved)
- `severity` — уровень критичности

### 4. Экспорт отчёта (Export)

**Компоненты:**
- `Button` — кнопка экспорта
- `ToastContext` — уведомление об успешном экспорте

**Функционал:**
- Экспорт в JSON/CSV
- Уведомление об успешном экспорте

## User Flow

1. Пользователь открывает Health Dashboard
2. Отображается общий статус системы и сетка компонентов
3. Пользователь может фильтровать компоненты по статусу/типу
4. При клике на компонент открывается модальное окно с деталями
5. В разделе Alerts отображаются активные предупреждения
6. Пользователь может экспортировать отчёт о состоянии системы

## Edge Cases

- **Нет данных о компоненте** — компонент отображается со статусом `unknown`
- **Прерванная проверка** — статус `unknown` с пометкой "проверка прервана"
- **Метрики недоступны** — метрики отображаются как `N/A`
- **Множественные ошибки** — ошибки отображаются в списке с количеством
- **Большие объёмы данных** — используется пагинация в DataTable
