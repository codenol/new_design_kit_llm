# Фича: Модальное окно статистики CVE

## Описание
При клике на блоки статистики (severity или status) нужно показывать модалку с CVE, которые соответствуют выбранной категории.

## Логика блоков

### Блок 1: Модальное окно (Dialog) для просмотра CVE по категории
- **Тип**: Dialog (модальное окно)
- **Триггер**: Клик на блок статистики
- **Контент**:
  - Заголовок с категорией (например, "Critical CVE" или "Unpatched CVE")
  - Список CVE в виде карточек, каждая содержит:
    - CVE ID
    - Severity (с StatusBadge)
    - Status (с StatusBadge)
    - CVSS Score (с цветовой индикацией)
    - Published Date
    - Описание
  - Кнопка "Закрыть"

### Блок 2: Связь с контекстом
- При клике на блок статистики нужно передавать категорию (severity или status) в Dialog
- Dialog должен открываться при клике на блок статистики
- Dialog должен закрываться по кнопке "Закрыть"
- Фильтрация CVE по выбранной категории (severity или status)

### Блок 3: Визуальная индикация
- Блоки статистики должны быть кликабельными (добавить курсор pointer и обработчик onClick)
- При наведении на блоки статистики — легкая подсветка (hover эффект)

## Дизайн-система
- Использовать существующие компоненты: Dialog, StatusBadge, Button
- Использовать дизайн-токены: `--surface-card`, `text-color-secondary`, `text-color-danger`, `text-color-warning`, `text-color-success`
- Следовать типографике и отступам дизайн-системы
- Карточки CVE в модалке должны использовать ту же стилистику, что и карточки в статистике

## Интеграция с существующим кодом
- Добавить новое состояние для хранения выбранной категории и списка CVE для отображения
- Добавить обработчики кликов на блоки статистики
- Реализовать фильтрацию CVE по выбранной категории
- Dialog должен быть стилизован в соответствии с дизайн-системой
