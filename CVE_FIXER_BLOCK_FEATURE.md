# Фича: Блок с данными о сотрудниках, исправивших CVE

## Описание

Добавить новый блок на страницу CVE Summary, который отображает информацию о сотрудниках, которые недавно исправляли уязвимости (CVE).

## Логику блоков

### Блок 1: Заголовок страницы
- H1: "Сводка по CVE"
- Краткое описание раздела

### Блок 2: Исправления от сотрудников (НОВЫЙ БЛОК)
- H2: "Исправления от сотрудников"
- Краткое описание: "Список сотрудников, которые недавно исправляли уязвимости"

#### Таблица исправлений
- Колонки:
  - **CVE ID** - идентификатор уязвимости (ссылка на CVE)
  - **Сотрудник** - имя сотрудника (ссылка на профиль)
  - **Дата исправления** - дата и время исправления
  - **Severity** - уровень серьезности (с цветовой индикацией)
  - **Статус** - статус исправления (Patched)
  - **CVSS Score** - оценка уязвимости

### Блок 3: Фильтры и поиск
- Поле поиска по названию CVE
- Фильтр по severity (Critical, High, Medium, Low)
- Фильтр по статусу (Unpatched, Patched, In Progress)
- Кнопка сброса фильтров

### Блок 4: Таблица CVE
- Колонки:
  - CVE ID
  - Severity (с цветовой индикацией)
  - Status
  - Description
  - CVSS Score
  - Published Date
  - Actions (кнопка деталей)

### Блок 5: Статистика
- Всего CVE
- По severity (Critical, High, Medium, Low)
- По статусу (Unpatched, Patched, In Progress)

## Дизайн-система
- Использовать компоненты из design-system/uikit:
  - DataTable для таблицы
  - StatusBadge для severity и status
  - Button для действий
- Стили из design-system/layout
- Цвета из design-system/tokens

## Расположение на странице
Новый блок размещается **после заголовка страницы (Блок 1)**, перед блоком фильтров и поиска.
